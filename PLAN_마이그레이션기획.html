Section 05

인터랙션 전략 — 모션그래픽 / WebGL / 스크롤 연동

모션그래픽 수준의 정교한 애니메이션. 단순 CSS 트윈이 아닌 After Effects급 퀄리티를 웹에서 구현.

인터랙션 4원칙:
1. 의미 기반 — 모든 모션에 이유가 있어야 함 (장식적 움직임 금지)
2. 절제된 밀도 — 한 뷰포트에 1개의 주요 인터랙션만. 동시다발 금지
3. 서사적 연결 — 스크롤이 내려가면서 '만남 → 연애 → 결혼 → 행복'의 여정을 은유
4. 모션그래픽 퀄리티 — Lottie/Rive로 AE 수준 애니메이션 구현. 단순 fade/slide 금지. 이징 커브, 오버슈트, 스태거 등 정교한 타이밍 설계

아이콘 규칙

컬러 이모지 사용 금지 — SVG 아이콘 전용

컬러 이모지는 프리미엄 톤을 저해합니다. 모든 아이콘은 다음 규칙을 따릅니다:

  • UI 아이콘: Lucide Icons (SVG, 모노라인 스타일) — 네비게이션, 버튼, 리스트 마커
  • 장식 아이콘: 커스텀 SVG 일러스트 — 카라꽃, 링, 리본 등 브랜드 오브제를 라인아트로
  • 애니메이션 아이콘: Lottie JSON — After Effects에서 제작한 마이크로 인터랙션 (호버, 로딩, 전환)
  • 컬러: 골드(stroke) 또는 아이보리(stroke) 단색. 그라데이션은 시그니처 포인트에만
  • 사이즈: 24px(인라인), 32px(카드), 48~64px(섹션 아이콘). 일관된 stroke-width: 1.5px

모션그래픽 기술 스택

도구 역할 적용 영역
Lottie (lottie-web) After Effects 제작 애니메이션을 JSON으로 export하여 웹에서 재생. 벡터 기반으로 해상도 무관. 스크롤 위치에 프레임 매핑 가능. 카라꽃 블룸, 텍스트 리빌, 아이콘 마이크로 인터랙션, 로딩 애니메이션
Rive 인터랙티브 스테이트 머신 기반 애니메이션. 마우스/스크롤/클릭에 실시간 반응하는 복합 모션. Lottie보다 인터랙션 자유도 높음. CTA 버튼 모션, 서비스 카드 호버 트랜지션, 프로세스 인포그래픽
GSAP + ScrollTrigger 스크롤 기반 타임라인 오케스트레이션. 정교한 이징 커브(elastic, back, expo), 스태거, 핀닝. Lottie/Rive 재생을 스크롤에 동기화하는 컨트롤러 역할. 전체 스크롤 플로우 제어, 커튼 리빌, 섹션 전환, Lottie 프레임 매핑
Three.js (WebGL) 실시간 3D 렌더링. PBR 머티리얼 + 환경맵으로 사실적 금속 질감. 모바일은 정적 이미지 폴백. 웨딩 링 3D, 골드 파티클 시스템

Lottie = 사전 제작된 정교한 모션 재생, Rive = 실시간 인터랙티브 모션, GSAP = 오케스트레이션 레이어. 세 도구가 각자의 강점 영역을 담당.

A. 히어로 — 카라꽃 블룸 (메인 시그니처)

Calla Lily Bloom on Scroll — Lottie Sequence

HOME 히어로 영역 (100vh)

페이지 로드 시 흰 카라꽃 봉오리가 화면 중앙에 위치. 스크롤을 내리기 시작하면 꽃잎이 서서히 펼쳐지며 활짝 핌. 꽃이 피는 동안 배경이 미세하게 밝아지고, 골드 파티클이 꽃술 주변에서 흩날림.

모션그래픽 접근:
• After Effects에서 카라꽃 개화 시퀀스 제작 (Shape Layer 기반, 60fps, 3초 분량)
• Bodymovin 플러그인으로 Lottie JSON export
• lottie-web의 goToAndStop(frame)을 GSAP ScrollTrigger에 연결 — 스크롤 위치(0~100vh)가 Lottie 프레임(0~90)에 1:1 매핑
• 꽃 주변 골드 파티클은 별도 Canvas 레이어에서 Three.js Points로 렌더링

F. 골드 파티클 앰비언트 (배경 레이어)

Ambient Gold Particles — WebGL Shader

HOME 히어로 + CTA 섹션 배경

극도로 절제된 골드 입자가 천천히 떠다님. 밀도: 화면 전체에 15~20개 수준. 마우스 근처에서 미세하게 반응(반발력). 눈에 띄지 않지만, 없으면 허전한 수준.

모션그래픽 접근:

• 파티클 형태: 원형 + 미세한 글로우 (fragment shader radial-gradient)
• 모바일: 파티클 5개로 축소, shader 복잡도 낮춤. 저사양 디바이스는 CSS radial-gradient 폴백.

G. 마이크로 인터랙션 (UI 요소)

UI Micro-interactions — Rive State Machine

전체 사이트 — 버튼, 카드, 네비게이션

모든 인터랙티브 요소에 모션그래픽 수준의 마이크로 인터랙션 적용. 단순 CSS hover가 아닌 상태 기반 모션.

모션그래픽 접근:
• CTA 버튼: Rive State Machine — idle(골드 테두리 미세 펄스) → hover(테두리 확장 + 배경 fill-in + 텍스트 컬러 전환) → press(스케일 0.97 + 리플 이펙트) → release(바운스백)
• 서비스 카드: hover 시 카드 내부 SVG 아이콘이 Lottie 애니메이션으로 재생 (1~2초 루프). 카드 그림자 depth 변화.
• 네비게이션: 메뉴 호버 시 밑줄이 SVG 드로잉으로 좌→우 그려짐. 페이지 전환 시 골드 라인 프로그레스.
• 스크롤 인디케이터: 커스텀 SVG 스크롤바. 진행도에 따라 골드 fill이 채워짐.

Phase 1 — 콘텐츠 정리 (현재 단계)

  1. 텍스트 마이그레이션 파일 생성

    기존 사이트 + 브로셔 텍스트를 페이지별로 정리한 마스터 콘텐츠 파일 작성. 리라이팅 필요 항목 표시.

  2. 에셋 목록 확정

    필요한 이미지/3D/텍스처 리스트업. 소스(스톡/AI생성/기존) 결정. 카라꽃 이미지 시퀀스 확보 방안.

  3. 컬러/타이포 토큰 확정

    CSS 변수로 디자인 토큰 정의. 최종 팔레트 승인.

Phase 2 — HOME 와이어프레임

  1. 정적 레이아웃

    Tailwind CDN으로 HOME 전체 섹션 배치. 텍스트/이미지 플레이스홀더.

  2. 히어로 인터랙션 프로토타입

    카라꽃 시퀀스 or 대체 효과 테스트. 스크롤 연동 검증.

  3. 모션 톤 확정

    GSAP 이징 커브, 전환 속도, 커튼 리빌 프로토타입으로 전체 모션 톤 결정.

결정 사항 (확정 + 미결)

확정 카라꽃 에셋
